meshPlatform

All endpoints on meshPlatforms.

A meshPlatform represents a cloud platform integration in meshStack. It defines how meshStack connects to and manages resources on cloud platforms like Azure, AWS, GCP, or Kubernetes clusters. Each meshPlatform is owned by a workspace and can be configured with specific access restrictions and marketplace availability.

Note: This API enables management for platform configuration around replication, but not yet for metering.

Note: This API does not yet support pricing on platforms.

Please refer to the meshPlatform Configurations section for details on how to configure meshPlatforms.

List meshPlatforms

Provides a paged list of meshPlatforms. Deleted meshPlatforms are not provided via this endpoint.

Depending on your permissions, you will see:

  • Admin users: All meshPlatforms in the system

  • Workspace users: Only meshPlatforms owned by their workspace, public platforms, or platforms they contribute to

Get meshPlatform

Get a single meshPlatform by its UUID. Deleted meshPlatforms are not provided via this endpoint. Please note that the configuration value is only filled for supported platform types. If the platform type is not supported, the configuration value will be empty. The supported platform types are listed here: meshPlatform Configurations.

Access is granted based on:

  • Platform ownership (workspace owns the platform)

  • Platform visibility (public platforms)

  • Platform contribution (workspace is a contributor)

  • Platform marketplace publication status

  • Admin permissions

Creating a meshPlatform requires proper platform configuration to be functional. After creation, you must:

Delete meshPlatform

Delete a meshPlatform by its UUID.

Warning

Deleting a meshPlatform has significant consequences:

  • Tenants: All associated meshTenants will be automatically deleted in meshStack, but the actual cloud resources remain untouched

  • Billing: Usage reports for the current month won’t be finalized and won’t be included in chargeback statements

  • Access: Users lose access to platform tenants via meshStack (but can still access them directly in the cloud platform)

  • Identifier: You cannot create a new meshPlatform with the same identifier in the same location

This operation cannot be undone. Ensure you have proper backups and have communicated the impact to affected users.

meshPlatform Configurations

A meshPlatform has a specific type and based on that type needs to be configured with individual configuration fields. You find example configurations with explanation annotations in this section.

With the current version of this API the following meshPlatform types are supported:

  • Azure Kubernetes Service (AKS)

  • Amazon Web Services (AWS)

  • Microsoft Azure (Azure)

  • Azure Resource Group (AzureRG)

  • Google Cloud Platform (GCP)

  • Kubernetes clusters (Kubernetes)

  • OpenShift clusters (OpenShift)

The example configuration provided here belongs to the specification of the respective meshPlatform object like so:

{
  "meshPlatform": {
    "metadata": { ... },
    "spec": {
      "displayName": "Example Platform",
      ...,
      "config": {
        "type": "$PLATFORM",
        "$PLATFORM": {  all platform specific properties  }
      }
    },
    "status": { ... }
  }
}

  • azurerg: Configuration for Azure Resource Group

  • gcp: Configuration for Google Cloud Platform

  • kubernetes: Configuration for Kubernetes clusters
